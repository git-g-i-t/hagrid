# 整理绘图脚本到 results 文件夹

根据你的要求，我将把所有用于生成报告图片的脚本统一整理到 `results/` 文件夹下，并确保生成的图片也保存在该目录中。

## 1. 移动与更新 `visualize_gradcam.py`
- **动作**：将根目录下的 `visualize_gradcam.py` 移动到 `results/visualize_gradcam.py`。
- **修改**：更新默认的 `output_dir` 参数，使其默认保存到 `results/gradcam`，并优化模块导入路径以支持从子目录运行。

## 2. 创建 `plot_val_test_curve.py`
- **动作**：在 `results/` 目录下新建此脚本。
- **功能**：
    - 读取 TensorBoard 日志。
    - 绘制验证集 (Val) 和测试集 (Test) 的 F1-Score 曲线。
    - 图片默认保存为 `results/val_test_curve.png`。

## 3. 目录结构预期
执行后，你的 `results/` 文件夹将变得井井有条：
```
results/
├── visualize_gradcam.py      # [脚本] 生成热力图
├── plot_val_test_curve.py    # [脚本] 生成性能曲线
├── plot_results.py           # [旧脚本] 现有绘图脚本
├── gradcam/                  # [图片] 存放热力图结果
├── val_test_curve.png        # [图片] 性能对比图
└── ...
```

## 4. 运行方式
整理后，建议在项目根目录运行这些脚本：
```powershell
# 生成热力图
python results/visualize_gradcam.py --checkpoint ...

# 生成曲线图
python results/plot_val_test_curve.py --log_dir ...
```
